## 个人信息保护法

### 核心原则

**1. 合法、正当、必要原则**
- 处理个人信息应当具有明确、合理的目的
- 应当与处理目的直接相关
- 采取对个人权益影响最小的方式

**2. 最小化原则**
- 限于实现处理目的的最小范围
- 不得过度收集个人信息

**3. 公开透明原则**
- 公开个人信息处理规则
- 明示处理的目的、方式和范围

**4. 准确性原则**
- 保证个人信息的质量
- 避免因个人信息不准确、不完整对个人权益造成不利影响

---

### AI 系统特殊要求

**1. 自动化决策**
- [ ] 保证决策的透明度和结果公平、公正
- [ ] 不得对个人在交易价格等交易条件上实行不合理的差别待遇
- [ ] 通过自动化决策向个人进行信息推送、商业营销，应当同时提供不针对其个人特征的选项

**2. 敏感个人信息**
- [ ] 生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等信息
- [ ] 应当取得个人的单独同意
- [ ] 应当具有特定的目的和充分的必要性
- [ ] 应当采取严格保护措施

**3. 个人信息跨境提供**
- [ ] 通过国家网信部门组织的安全评估
- [ ] 按照国家网信部门的规定经专业机构进行个人信息保护认证
- [ ] 与境外接收方订立标准合同

## 金融数据安全规定

### 数据分类分级

**1. 核心数据**
- 国家金融安全相关数据
- 重要金融基础设施数据
- 涉及国计民生的重要金融数据

**2. 重要数据**
- 大额存贷款数据
- 重要客户信息
- 重要交易数据
- 风险管理数据

**3. 一般数据**
- 普通客户信息
- 一般业务数据
- 内部管理数据

---

### 安全技术要求

**1. 数据加密**
- [ ] 传输加密（TLS 1.2+）
- [ ] 存储加密（AES-256）
- [ ] 密钥管理（HSM/KMS）

**2. 访问控制**
- [ ] 身份认证（多因素认证）
- [ ] 权限管理（最小权限原则）
- [ ] 访问审计（完整日志）

**3. 数据脱敏**
- [ ] 开发测试环境脱敏
- [ ] 生产环境查询脱敏
- [ ] 数据共享脱敏

**4. 安全审计**
- [ ] 操作日志完整记录
- [ ] 异常行为检测
- [ ] 审计报告生成

### AI生成代码的典型特征（黑名单）

## 🔧 代码质量改进措施

### 1. 代码质量检查清单（强制执行）

**提交前必须检查**：
- [ ] docstring ≤ 3行（复杂逻辑≤5行）
- [ ] 无列表化配置说明
- [ ] 无冗余解释（基础概念）
- [ ] 匹配项目代码风格
- [ ] 至少3次自测（不同场景）
- [ ] PR描述完整（问题+方案+测试）
- [ ] 敏感信息检查（API key/密码）

### 2. 代码风格匹配流程

**步骤**：
1. **阅读项目现有代码**：至少3个同类文件
2. **提取风格特征**：
   - 注释密度
   - docstring格式
   - 变量命名习惯
   - 代码组织方式
3. **模仿风格**：
   - 使用相同的注释密度
   - 使用相同的docstring格式
   - 使用相同的变量命名
4. **自测验证**：
   - 功能测试（至少3个场景）
   - 边界测试（异常输入）
   - 集成测试（与现有代码交互）

### 3. 代码质量门禁

**禁止提交**：
- docstring > 5行
- 配置项用列表枚举
- 重复解释基础概念
- 缺乏自测证明

**必须包含**：
- 功能测试截图/日志
- 边界测试用例
- 与现有代码的对比截图

### 4. AI特征识别自检

**自查问题**：
1. 这段代码是否过于"完美"？
2. 注释是否多于必要？
3. docstring是否像教科书？
4. 是否缺乏人类的不完美？

**如果答案是"是" → 重写**
